Chapter 1

What Is This Stack?

The plain English version of what you've built and why it matters.

You've built an automation system that creates social media posts, sends them to the right platforms at the right times, tracks results, and even sends review requests — all without you lifting a finger.

Think of it like hiring a social media team that works 24/7, never takes a holiday, and costs less than a tenner a month to run. The "team" is made up of software tools that talk to each other automatically.

Here's the deal: you own 5 different brands. Each one needs a social media presence. Posting manually to 10+ platforms across 5 brands would take hours every day. Instead, you've built a pipeline where events in your business automatically trigger social media posts.

What Each Tool Does

n8n — The Brain

The automation engine. Think of it as your robot assistant that runs workflows 24/7. It listens for events (like a new lead), then runs a series of steps automatically. Self-hosted means you own the data and there are no per-task fees.

Claude AI — The Writer

Writes all your social media posts. You give it context — what happened, which brand, which platform — and it creates content that sounds human and matches each platform's style. A LinkedIn post reads differently from a TikTok caption, and Claude handles that automatically.

Airtable — The Database

Your CRM and tracking system. Every post, every lead, every result gets logged here. Think of it as a really smart spreadsheet that the automation can read from and write to. You can open it anytime to see what's been posted and how it performed.

Twilio — The SMS Sender

Sends text messages. Used for Astra Removals to request Google reviews after a move is completed. A customer finishes their move, and 48 hours later they get a friendly SMS asking for a review. Costs about 2p per message.

Slack — The Alert System

Where you get notified if a post needs human review. When Claude writes something it isn't fully confident about (score below 75), it sends you a message in Slack so you can approve, edit, or reject it before it goes live.

Meta Graph API — FB + IG Publishing

The official way to publish posts to Facebook and Instagram automatically. One API handles both platforms because Meta owns them both. You'll need a Facebook Page and a linked Instagram Business account for each brand that uses them.

LinkedIn API — B2B Content

Publishes posts and articles to LinkedIn company pages. Mainly used for Magna Park (B2B storage marketing) and Rick.AI (thought leadership). Requires admin access to each company page.

X / Twitter API — Threads + Posts

Publishes tweets and threads. Used for CoachMentor (match day threads) and Rick.AI (build logs). Needs 5 separate API keys — it's the fiddliest setup of all the platforms.

Pinterest API — Pins + Boards

The easiest platform API of all. Publishes pins to themed boards. Great for Prime Haul (moving tips), CoachMentor (coaching drills), and Rick.AI (tech infographics). Pins have the longest content lifespan of any platform — they keep getting traffic for months.

YouTube Data API — Video Publishing

Uploads videos and manages your YouTube channels. Used for CoachMentor coaching content. Set up through Google Cloud Console using OAuth.

TikTok API — Short Video

Publishes short-form video content. Used for CoachMentor and Prime Haul. The API requires an application review that takes 1–2 weeks, so apply early. Until approved, you'll post manually using saved captions.

Google Business Profile — Local SEO

Publishes posts to your Google Business listing — the box that shows up when someone searches for your business. Used for Astra Removals and Magna Park. Needs verification (postcard or phone call from Google).

Cross-Brand Magic
If someone is moving in Dorset, the post automatically mentions Magna Park storage. One lead, two businesses promoted. This is the astra-magna-workflow.n8n.json file — a single workflow that serves both brands.
The Confidence Gate
Claude scores its own work 0–100. Think of it like a self-review. Below 75 means "I'm not fully sure this is great" — it sends the post to your Slack for you to approve, edit, or reject. Above 75, it publishes automatically. This means you only see posts that need attention.
Smart Scheduling
Posts don't go out immediately. Research shows removal-related content performs best on Tuesday to Thursday evenings (around 7:30pm) and Saturday mornings (around 10am). The system queues posts for these windows.

Copyright Guard — Why 80%?
CoachMentor teaches FA coaching badges, which means there's a risk of accidentally reproducing FA or UEFA licensed materials. The copyright originality threshold is set to 80% — stricter than the 75% confidence gate used by other brands. If content looks too close to existing copyrighted material, it gets flagged for your review.
The DM Funnel Logic
Not everyone who engages is a potential customer. Claude analyses profiles and engagement patterns to work out if someone is a coach (worth a direct message), a parent (add to ad audience), or a player (just tag and move on). Only coaches get the direct outreach — no spamming random people.

Phase 2: API Tokens

Active time: ~2 hours

Each platform needs API tokens — think of these as passwords that let n8n post on your behalf. Some are easy to get, others are fiddly. Here they are ranked from easiest to hardest:

1
Pinterest
Just generate a token in the Developer Portal. One click. Done.
2
YouTube / Google
OAuth flow in Google Cloud Console. Create a project, enable the API, generate credentials. About 15 minutes.
3
Facebook / Instagram
Create a Meta Developer App → Generate a short-lived token → Exchange for a long-lived token (lasts 60 days) → Exchange again for a permanent page token. About 20 minutes.
4
LinkedIn
Create an app in LinkedIn Developer Portal, then go through an OAuth flow. The token lasts 60 days and needs refreshing.
5
X / Twitter
You need FIVE separate keys: Bearer Token, API Key, API Secret, Access Token, Access Secret. All generated in the Developer Portal but it's the most keys of any platform.
6
TikTok
Application review required (1–2 weeks). You can't generate tokens until approved. Apply early and work on other platforms while waiting.

Where Do Tokens Go?

Each brand has an env template file that lists every token it needs. These are in social-setup/env-templates/:

env-templates/ env-template-astra.env # FB, IG, GMB, Twilio, Airtable env-template-coachmentor.env # FB, IG, TikTok, YouTube, X, Airtable, ManyChat env-template-magna.env # FB, IG, LinkedIn, GMB, Airtable env-template-primhaul.env # FB, IG, TikTok, Pinterest, Airtable env-template-rickai.env # X, LinkedIn, IG, FB, Pinterest, Airtable

Open each file, fill in the values, and paste them into n8n's Environment Variables settings (Settings → Environment Variables in n8n). The comments in each file explain where to find each value.

Pro Tip: The Anthropic Key is Shared
The ANTHROPIC_API_KEY (for Claude AI) and AIRTABLE_API_TOKEN are the same across all brands. You only need to set them once.

Step 2: Import Your Workflows

You have 4 workflow files ready to import. In n8n:

  3. Select one of your .n8n.json files
  4. The entire workflow appears with all nodes connected
  5. Add your credentials (tokens) to each node that needs them

Step 3: Add Credentials

Each workflow has nodes that connect to external services (Facebook, Airtable, Claude, etc.). Click on any node, and if it needs credentials, you'll see a dropdown. Create new credentials by pasting in the API tokens from your env template.

Step 4: Test with Sample Data

Before going live, test each workflow manually:

  1. Click Execute Workflow in n8n
  2. If it uses a webhook, send a test request (n8n gives you a test URL)
  3. Watch each node execute — green = success, red = error
  4. Check the output of each node to confirm the data flows correctly

Phase 4: Airtable Setup

Active time: ~1 hour

  1. Create a free Airtable account at airtable.com
  2. Create 5 bases (one per brand): Astra Removals, Magna Park, CoachMentor, Prime Haul, Rick.AI
  3. Add tables using the schema files — each JSON file in airtable-schemas/ lists the exact table names, column names, and field types you need
  4. Generate an API token at airtable.com/create/tokens — give it read/write access to all 5 bases
  5. Copy each base ID from the Airtable URL (it looks like appXXXXXXXXXXXX) into your env templates
What Are Schema Files?
The schema files don't create the tables automatically — they're a blueprint showing you exactly what to create. Open the JSON file, and you'll see each table name and its fields listed. Create them manually in Airtable by clicking "Add Table" and "Add Field". It takes about 10–15 minutes per brand.

Chapter 8

Troubleshooting & FAQ

Common problems, their solutions, and answers to the questions you're probably asking.

Common Issues

"My post didn't go live"

▼

Most likely cause: The confidence score was below 75, so it was sent to Slack for your review instead of publishing automatically.

Check: Open Slack and look for a message from n8n with the post content and an Approve/Reject option.

Other causes:

  • The API token expired (Meta tokens expire every 60 days)
  • The platform's API rate limit was hit (rare, but possible with high volume)
  • The workflow is paused/inactive in n8n

"n8n workflow failed with a red error"

▼

First step: Click on the red node to see the error message. Common errors:

  • "401 Unauthorized" — Your API token is wrong or expired. Regenerate it.
  • "429 Too Many Requests" — You've hit the platform's rate limit. The workflow will retry automatically.
  • "400 Bad Request" — The data format is wrong. Check the input to the failing node.

If you can't work it out, copy the error message and ask Claude Code to diagnose it.

"Token expired" error

▼

Meta (Facebook/Instagram) tokens expire every 60 days. You need to exchange them for a new long-lived token. The process:

  1. Go to the Meta Graph API Explorer
  2. Generate a new short-lived User Token
  3. Exchange it for a long-lived token using the API
  4. Then exchange that for a permanent Page Token
  5. Update the token in your n8n credentials

Ask Claude Code to walk you through this — it can do the API calls for you.

LinkedIn tokens also expire every 60 days. Refresh via the LinkedIn Developer Portal.

Pinterest, X, and YouTube tokens don't expire (or auto-refresh).

"TikTok isn't posting"

▼

TikTok API approval takes 1–2 weeks. Until approved, the workflow saves the caption and hashtags to Airtable so you can post manually. Check your TikTok Developer Portal for approval status.

Once approved, update your env variables with the TikTok tokens and the workflow will start posting automatically.

"The content doesn't sound right"

▼

The quality of the content depends on the system prompt given to Claude. Each workflow has a "Generate Content" node with detailed instructions about tone, style, and brand voice.

To adjust: open the workflow in n8n, find the Claude/AI node, and edit the system message. You can also ask Claude Code to refine the prompt for you.

Can I edit posts before they go live? ▼

Yes. Posts with a confidence score below 75 are sent to your Slack workspace for review. You can approve them as-is, edit the text, or reject them entirely. You can also raise the threshold to see more posts before they publish (e.g., set it to 90 to review almost everything).

What if I want to pause automation for a brand? ▼

In n8n, each workflow has an on/off toggle. Click it to deactivate. The workflow stops running immediately. Click again to reactivate. No data is lost — it just pauses.

If you're using n8n Cloud, it runs on their servers (not your computer), so your internet doesn't matter. If self-hosted, the VPS runs independently. Workflows queue up and retry if a platform is temporarily unreachable.
